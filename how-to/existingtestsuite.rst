如何将 pytest 与现有测试套件一起使用
==============================================

Pytest 可以与大多数现有测试套件一起使用，但它的行为与其他测试运行器（如 Python 的默认 unittest 框架）不同。

在使用本节之前，你需要 :ref:`安装 pytest <getstarted>`。

使用 pytest 运行现有测试套件
---------------------------------------------

假设你想为某处的现有仓库做贡献。在使用某种版本控制将代码拉入你的开发空间并（可选地）设置虚拟环境后，你将需要运行：

.. code-block:: bash

    cd <repository>
    pip install -e .  # 依赖于环境的替代方法包括
                      # 'python setup.py develop' 和 'conda develop'

在你的项目根目录中。这将在 site-packages 中设置一个指向你代码的符号链接，允许你在测试针对它运行时编辑你的代码，就像它已安装一样。

以开发模式设置你的项目让你不必每次想要运行测试时都重新安装，而且比摆弄 sys.path 来将测试指向本地代码更不容易出错。

还可以考虑使用 :ref:`tox <use tox>`。
