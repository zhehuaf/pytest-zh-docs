测试的剖析
=================

最简单地说，测试是为了查看特定行为的结果，并确保该结果与你的预期一致。行为是不能被实证测量的东西，这就是为什么编写测试可能具有挑战性。

"行为"是某个系统对特定情况和/或刺激作出反应的方式。但*如何*或*为什么*做某件事并不像*做了什么*那么重要。

你可以将测试分解为四个步骤：

1. 准备 (Arrange)
2. 执行 (Act)
3. 断言 (Assert)
4. 清理 (Cleanup)

准备是我们为测试准备一切的地方。这意味着除了"执行"之外的几乎所有事情。它是在排列多米诺骨牌，以便执行可以在一个改变状态的步骤中完成其工作。这可以意味着准备对象、启动/停止服务、将记录输入数据库，甚至像定义要查询的 URL、为尚不存在的用户生成一些凭证，或者只是等待某个进程完成。

执行\是启动我们想要测试的\行为\的单一、改变状态的动作。这种行为是执行被测系统 (SUT) 状态改变的行为，正是这个改变后的状态，我们可以查看以对行为做出判断。这通常采用函数/方法调用的形式。

断言\是我们查看该结果状态并检查尘埃落定后它是否如我们所预期的那样。它是我们收集证据来说明行为是否符合我们预期的地方。测试中的 ``assert`` 是我们进行测量/观察并对其做出判断的地方。如果某物应该是绿色的，我们会说 ``assert thing == "green"``。

清理\是测试自行清理的地方，以便其他测试不会被它意外影响。

归根结底，测试最终是执行和断言\步骤，准备\步骤只提供上下文。行为\存在于执行\和断言\之间。
